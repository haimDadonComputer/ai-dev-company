# Project Definition

## Project Name

תשתית בסיס לאפליקציות פרויקטליות

## Project Goal

ליצור בסיס רב-שימושי שממנו ניתן להקים פרויקטים חדשים בלי לכפות עליהם ממשק גרפי או זהות חזותית קיימת. הבסיס מספק תשתיות שרת, אימות, העלאת קבצים, context ניהולי וקובץ JSON גלובלי לפרטי אתר.

## Target Audience

- מפתח שמעתיק ומרחיב את התשתית עבור פרויקט חדש.
- AI Manager וסוכנים שמנהלים שינויי מערכת לפי מקורות אמת.
- מבקר לא מחובר שצורך מידע ציבורי דרך API.

## Roles

- `guest`: גישה למידע ציבורי בלבד.
- `admin`: גישה לנתיבי API ניהוליים כאשר פרויקט נגזר מספק ממשק מתאים.

## Modules

- Authentication API
- Users
- Global site config JSON
- Media uploads API
- Public settings API
- Manager Bootstrap

## Business Rules

- בסיס המערכת אינו כולל מסכי התחברות או ממשק ניהול גרפי.
- כל פרויקט נגזר בונה את הממשק הגרפי שלו מאפס לפי הצורך.
- פרטי אתר בסיסיים נשמרים ב-`src/app/site-config.json`.
- הקובץ הגלובלי כולל: לוגו אתר, שם אתר, תיאור אתר, סלוגן, favicon, טלפון עסקי, כתובת, אינסטגרם, פייסבוק ו-WhatsApp.
- משתמש מנהל התחלתי יווצר במסד הנתונים בשם `admin`.
- הסיסמה ההתחלתית תהיה `123123`, תישמר כגיבוב מאובטח ולעולם לא כטקסט גלוי.
- סיסמה חדשה תקינה היא סיסמה עם יותר מ-6 תווים.
- JWT יישמר ב-HttpOnly Cookie בלבד ולא ב-localStorage.
- העלאת קבצים תתבצע באמצעות בחירת קובץ מהמכשיר ושמירתו בשרת.
- גודל קובץ מרבי הוא 5MB.
- יתקבלו רק סוגי קבצים שהוגדרו במפורש ב-allowlist.
- שמות קבצים בשרת יהיו אקראיים ולא יתבססו על שם הקובץ שסיפק המשתמש.
- כל סוד, לרבות מפתח JWT ופרטי MySQL, יוגדר ב-`.env`.
- כל פרויקט נגזר חייב להגדיר Design Profile ייחודי לפני פריסה לפרודקשן.
- שינוי צבעים בלבד אינו נחשב זהות חזותית חדשה.

## Architecture Decisions

- Backend: Node.js, Express ו-TypeScript.
- Database: MySQL עם `mysql2` ו-SQL ישיר בלבד.
- Frontend base: `index.html` יחיד, Vanilla TypeScript, קובץ CSS מינימלי וטעינת `site-config.json`.
- Tables: כל טבלה בפרויקט נגזר תיבנה עם `ag-grid-community` בלבד.
- Password hashing: `crypto.scrypt` המובנה ב-Node עם salt אקראי.
- File upload: הזרמה בינארית לשרת ללא ספריית multipart חיצונית.
- Storage: קובצי מדיה בדיסק השרת ומטא-דאטה במסד הנתונים.
- Git אינו חלק מתהליך העבודה הנוכחי בהתאם להוראת `AGENTS.md`.
