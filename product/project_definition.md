# Project Definition

## Project Name

תשתית בסיס לאפליקציות ניהול

## Project Goal

ליצור בסיס רב-שימושי שממנו ניתן להקים פרויקטים חדשים, עם מערכת הזדהות
למנהל מערכת, ניהול הגדרות אתר והעלאת קבצים מאובטחת.

## Target Audience

- מנהל מערכת שמגדיר ומתחזק את פרטי האתר.
- מבקר לא מחובר שצורך מידע ציבורי שהוגדר במערכת הניהול.
- מפתח שמעתיק ומרחיב את התשתית עבור פרויקט חדש.

## Roles

- `guest`: גישה למידע ציבורי בלבד.
- `admin`: התחברות לממשק הניהול, ניהול הגדרות כלליות והעלאת קבצים.

## Modules

- Authentication
- Users
- Admin shell
- General settings
- Media uploads
- Public settings

## Business Rules

- משתמש מנהל התחלתי ייווצר במסד הנתונים בשם `admin`.
- הסיסמה ההתחלתית תהיה `123123`, תישמר כגיבוב מאובטח ולעולם לא כטקסט גלוי.
- מנהל שמתחבר עם הסיסמה ההתחלתית יחויב להחליף אותה לפני גישה לניהול.
- JWT יישמר ב־HttpOnly Cookie בלבד ולא ב־localStorage.
- העלאת קבצים תתבצע באמצעות כפתור ובחירת קובץ מהמכשיר.
- גודל קובץ מרבי הוא 5MB.
- יתקבלו רק סוגי קבצים שיוגדרו במפורש ב־allowlist.
- שמות קבצים בשרת יהיו אקראיים ולא יתבססו על שם הקובץ שסיפק המשתמש.
- עמוד הניהול `כללי` יאפשר לערוך לוגו, שם אתר, סלוגן, טלפון, כתובת,
  WhatsApp, Instagram, אימייל ותמונות כלליות.
- כל שינוי בהגדרות יישמר עם חותמות זמן ומזהה המשתמש שביצע אותו.
- כל סוד, לרבות מפתח JWT ופרטי MySQL, יוגדר ב־`.env`.
- כל פרויקט נגזר חייב להגדיר Design Profile ייחודי לפני פריסה לפרודקשן.
- שינוי צבעים בלבד אינו נחשב זהות חזותית חדשה.
- לפחות ארבעה ממדי עיצוב מהותיים חייבים להשתנות בין פרויקטים.

## Architecture Decisions

- Backend: Node.js, Express ו־TypeScript.
- Database: MySQL עם `mysql2` ו־SQL ישיר בלבד.
- Frontend: SPA עם Vanilla HTML, Vanilla CSS ו־TypeScript, בכיוון RTL.
- Password hashing: `crypto.scrypt` המובנה ב־Node עם salt אקראי.
- File upload: הזרמה בינארית לשרת ללא ספריית multipart חיצונית.
- Storage: קובצי מדיה בדיסק השרת ומטא-דאטה במסד הנתונים.
- Design Architecture: התנהגות הרכיבים משותפת, אך טוקנים, פריסה וזהות
  חזותית מוגדרים ב־Design Profile פרויקטלי.
- Git אינו חלק מתהליך העבודה הנוכחי בהתאם להוראת קובץ `AGENTS.md` הראשי.
